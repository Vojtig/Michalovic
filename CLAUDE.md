## Build

```bash
# Compile all JSX files in src/ to JS (must run before committing)
npm run build
```

This runs Babel on `src/*.jsx` → `src/*.js`. The HTML pages load the compiled `.js` files, not the `.jsx` sources. **Always run `npm run build` after editing any `.jsx` file.**

## Architecture

Static React site deployed via FTP to `michalovic.endora.site`. No bundler — React is loaded from CDN, JSX is pre-compiled by Babel CLI locally.

**Pattern for each app:**
1. `src/<name>.jsx` — source (React + JSX)
2. `src/<name>.js` — compiled output (generated by `npm run build`)
3. `<name>.html` — loads compiled `.js` via `<script src="src/<name>.js">`

React is available globally as `window.React`/`window.ReactDOM` (CDN). Components use `const { useState, useEffect } = React;` at the top instead of imports.

**Adding a new app:**
1. Create `src/<name>.jsx` with a root `App` component and `ReactDOM.createRoot(...).render(<App />)` at the bottom
2. Create `<name>.html` following the pattern of `shopping-list.html`
3. Add an `<a href="<name>.html">` card in `src/land.jsx` inside the `.apps-grid` div
4. Run `npm run build`

**Shared `style.css`** — all pages use the same stylesheet. Landing page styles and per-app styles coexist in one file.

**localStorage** — data is per-domain, so all apps on the same domain share the same localStorage namespace. Use unique keys (e.g. `shoppingList`, `shoppingHistory`).

## Deployment

Push to `master` → GitHub Actions automatically FTP-deploys to the server. The workflow excludes `tests/`, `node_modules/`, `.git*`, `.babelrc`, and `package*.json` — only HTML, CSS, and compiled JS in `src/` are uploaded.

Tests live in `tests/` and are never deployed. Test HTML files are opened directly in a browser; test JS files can be run with Node.
